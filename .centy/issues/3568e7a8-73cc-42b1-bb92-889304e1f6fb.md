# fix: comply with jsx-classname/require-classname rule

## Overview

The `jsx-classname/require-classname` rule is currently disabled globally in `eslint.config.mjs` because the project uses inline styles.

## Goal

Remove the override and comply with the rule by replacing inline styles with Tailwind/CSS classes throughout the codebase.

## Tasks

- [ ] Audit all components using inline styles
- [ ] Replace inline styles with equivalent Tailwind utility classes or CSS modules
- [ ] Remove `'jsx-classname/require-classname': 'off'` from `eslint.config.mjs`

## Acceptance Criteria

- [ ] No inline style props remain in JSX (except where dynamically computed values are unavoidable)
- [ ] ESLint passes without the override
