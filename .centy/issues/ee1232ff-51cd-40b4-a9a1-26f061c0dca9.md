# feat: show latest publish date per package

## Overview

Each package row currently shows version and download count. Adding the latest publish date gives users a quick signal of how actively maintained a package is.

## Tasks

- [ ] Fetch `https://registry.npmjs.org/<pkg>` and read `time.modified` or `time[latest]`
- [ ] Display the date in each `PkgRow` in a human-readable relative format (e.g. "3 days ago")
- [ ] Batch the registry calls alongside `fetchStats()` to avoid extra waterfalls
- [ ] Handle missing or malformed date fields gracefully

## Acceptance Criteria

- [ ] Each package row shows a publish date
- [ ] Date falls back to `—` if unavailable
- [ ] No additional network waterfalls introduced
