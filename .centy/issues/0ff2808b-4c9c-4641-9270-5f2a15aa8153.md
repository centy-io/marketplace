# fix: comply with default/no-hardcoded-urls rule

## Overview

The `default/no-hardcoded-urls` rule is currently disabled in two places in `eslint.config.mjs`:

- Globally (npm registry URLs treated as intentional constants)
- In `src/mocks/**` (MSW mock files)

## Goal

Remove both overrides and comply with the rule by extracting hardcoded URLs into named constants or environment-based configuration.

## Tasks

- [ ] Identify all hardcoded URLs in the codebase (npm registry URLs, MSW handler URLs, etc.)
- [ ] Extract them into named constants in a dedicated config or constants file
- [ ] Update MSW mock handlers to reference those constants
- [ ] Remove `'default/no-hardcoded-urls': 'off'` from both override blocks in `eslint.config.mjs`

## Acceptance Criteria

- [ ] No bare URL string literals remain in source files
- [ ] All URLs are defined as named, exported constants
- [ ] ESLint passes without either override
