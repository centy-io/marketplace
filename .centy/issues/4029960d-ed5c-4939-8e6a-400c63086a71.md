# Filter package list to only show centy-\* packages

The marketplace currently lists all packages in the \`FALLBACK\` array, including \`@centy-io/assets\` which does not follow the \`centy-_\` naming convention. The page should only display packages whose name matches the \`centy-_\` pattern.

## Problem

In \`src/app/page.tsx\`, the \`FALLBACK\` array includes \`@centy-io/assets\` (scoped package name: \`assets\`), which does not start with \`centy-\`. This inconsistency means non-centy-prefixed packages appear in the listing.

## Expected Behavior

Only packages whose name starts with \`centy-\` (or whose unscoped name starts with \`centy-\`) should be listed:

- \`centy\` — base CLI (exact match, keep)
- \`centy-plugin-persona\` — matches \`centy-\*\` ✓
- \`@centy-io/centy-installer\` — unscoped name \`centy-installer\` matches ✓
- \`@centy-io/centy-installer-\*\` platform packages — all match ✓
- \`@centy-io/assets\` — unscoped name \`assets\` does **not** match ✗ → **remove**

## Tasks

- Remove \`@centy-io/assets\` from the \`FALLBACK\` array in \`src/app/page.tsx\`
- Remove the \`assets\` group type from the \`Pkg\` interface and related rendering logic (the \`ASSETS\` section and \`SectionHeader\` call)
- Update the \`description\` metadata and any header copy that references the total package count if needed
