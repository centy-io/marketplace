# fix: comply with no-restricted-syntax rule in Next.js config files

## Overview

The `no-restricted-syntax` rule is currently disabled for `next.config.ts` and `src/instrumentation.ts` in `eslint.config.mjs` because these framework entry points require patterns (e.g., `process.env` access) that the rule restricts.

## Goal

Understand which specific syntax patterns are restricted and refactor the affected files to avoid them, or upstream an exclusion to `eslint-config-agent` for known framework files.

## Tasks

- [ ] Identify which restricted syntax patterns are used in `next.config.ts` and `src/instrumentation.ts`
- [ ] Refactor to avoid the restricted patterns where possible
- [ ] If the patterns are genuinely required by the Next.js framework, open an issue or PR in `eslint-config-agent` to exclude them for framework entry points
- [ ] Remove `'no-restricted-syntax': 'off'` from `eslint.config.mjs`

## Acceptance Criteria

- [ ] `no-restricted-syntax` runs without being suppressed in this repo
- [ ] ESLint passes without the override
