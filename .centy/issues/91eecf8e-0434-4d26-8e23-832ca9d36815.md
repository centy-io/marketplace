# feat: download period toggle (week / month / all-time)

## Overview

The page currently shows only last-month download counts. Allowing users to switch between periods gives a fuller picture of package traction.

## Periods to support

- Last week (`/downloads/point/last-week/<pkg>`)
- Last month (`/downloads/point/last-month/<pkg>`) — current default
- All time (`/downloads/range/2000-01-01:<today>/<pkg>`)

## Tasks

- [ ] Add a client-side period toggle component (week / month / all-time)
- [ ] Fetch stats for all three periods at build time (server component) and pass to client
- [ ] Animate the count transition when switching periods
- [ ] Persist the selected period in `localStorage` or URL search param

## Acceptance Criteria

- [ ] Toggle switches displayed counts without a page reload
- [ ] Selected period is preserved on refresh
- [ ] All-time count is the sum across the full date range
