# Testing: Add MSW service and Playwright screenshot tests

## Overview

Set up [MSW (Mock Service Worker)](https://mswjs.io/) for API mocking in tests, and add [Playwright](https://playwright.dev/) for screenshot (visual regression) testing.

## Goals

- Intercept and mock HTTP requests in tests using MSW so tests are fast and deterministic
- Capture and compare page/component screenshots with Playwright to catch unintended visual regressions

## Tasks

- [ ] Install and configure MSW (`msw`)
  - [ ] Define request handlers for all external APIs used by the marketplace
  - [ ] Integrate MSW with Next.js dev server (browser service worker) and test environment (Node.js server)
- [ ] Install and configure Playwright (`@playwright/test`)
  - [ ] Add `playwright.config.ts` with appropriate base URL, browser targets, and screenshot settings
  - [ ] Write screenshot tests for key pages (home, package detail, search results)
  - [ ] Store baseline screenshots in version control under `tests/screenshots/`
- [ ] Wire MSW into Playwright tests so network calls are intercepted during screenshot runs
- [ ] Add a `test:screenshots` npm script and document how to update baselines

## Acceptance Criteria

- [ ] `npm run test:screenshots` runs Playwright and captures screenshots with no live network calls
- [ ] A diff is produced when a visual regression is introduced
- [ ] MSW handlers cover all API endpoints used in tested pages
- [ ] CI workflow runs screenshot tests and fails on regressions
