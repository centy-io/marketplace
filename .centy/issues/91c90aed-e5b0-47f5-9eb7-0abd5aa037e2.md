# feat: auto-discover centy-\* packages from npm registry

## Overview

The package list in `src/app/page.tsx` is hardcoded in the `FALLBACK` array. Any new package requires a manual code change. Instead, discover packages automatically from the npm registry search API.

## Tasks

- [ ] Query `https://registry.npmjs.org/-/v1/search?text=centy&size=50` at build time
- [ ] Parse results and map to the `Pkg` shape (name, version, description, group)
- [ ] Infer `group` from package name/scope heuristics (e.g. `@centy-io/*-installer-*` → `platform`)
- [ ] Keep `FALLBACK` as a safety net if the registry call fails
- [ ] Revalidate on a schedule (ISR or cron-triggered rebuild)

## Acceptance Criteria

- [ ] Adding a new `centy-*` package to npm is reflected on the site without a code change
- [ ] The page still renders correctly if the registry API is unreachable
