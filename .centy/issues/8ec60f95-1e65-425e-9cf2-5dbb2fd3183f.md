# Fix pre-push hook reliability issues

## Background

During a push session, the pre-push hooks (lint-staged, cspell, knip, commitlint) failed in sequence due to several pre-existing misconfigurations. These were patched ad-hoc but the underlying problems should be properly addressed.

## Issues Found

### 1. cspell not using project word list

The `spell` / `spell:check` npm scripts and the lint-staged cspell command did not pass `--config .centy/cspell.json`, so the project word list (e.g. `centy`, `Centy`) was ignored when checking files outside `.centy/`.

**Workaround applied:** added `--config .centy/cspell.json` to both the npm scripts and the lint-staged command.

### 2. ESLint crashes on JS config files (FlatCompat + eslint-plugin-react circular reference)

Running `eslint --fix` on `commitlint.config.js` causes a `TypeError: Converting circular structure to JSON` inside `@eslint/eslintrc`'s config validator when loading `next/core-web-vitals` via `FlatCompat`.

**Workaround applied:** changed the lint-staged ESLint glob from `*.{ts,tsx,js,jsx}` to `*.{ts,tsx,jsx}` to skip plain `.js` files. Root cause (FlatCompat + eslint-plugin-react compatibility with ESLint 9/10) is unresolved.

### 3. `autoprefixer` listed as unused by knip

`autoprefixer` was in `devDependencies` but not used directly — Tailwind CSS v4 handles prefixing via `@tailwindcss/postcss`.

**Workaround applied:** removed `autoprefixer` from `devDependencies`.

### 4. `@eslint/eslintrc` falsely flagged as unused by knip

Knip does not detect the import of `@eslint/eslintrc` in `eslint.config.mjs` (used for `FlatCompat`).

**Workaround applied:** added `knip.config.ts` with `ignoreDependencies: ['@eslint/eslintrc']`.

### 5. lint-staged single-brace glob warning

Using `*.{js}` in lint-staged triggers a warning and gets auto-normalized to `*.js`.

## Tasks

- [ ] Investigate root cause of ESLint FlatCompat + eslint-plugin-react circular reference and apply a proper fix (e.g. migrate ESLint config away from `FlatCompat`, upgrade `eslint-plugin-react`, or pin compatible versions)
- [ ] Move cspell config to a root-level `cspell.json` (or add a root config that imports `.centy/cspell.json`) so all tools pick it up without explicit `--config` flags
- [ ] Verify knip correctly detects `@eslint/eslintrc` usage and remove the `ignoreDependencies` workaround if possible
- [ ] Clean up lint-staged globs to avoid single-brace warnings
- [ ] Add a CI job that runs all pre-push checks so regressions are caught before they reach the hook
