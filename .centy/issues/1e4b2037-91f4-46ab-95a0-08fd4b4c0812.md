# CI: Add GitHub Action to build and deploy to Cloudflare Pages twice daily

## Summary

Add a GitHub Actions workflow that automatically builds the Next.js project and deploys it to Cloudflare Pages at http://marketplace.centy.io/ on a scheduled basis twice per day.

## Requirements

- **Trigger**: Scheduled cron job running twice per day (e.g. 06:00 and 18:00 UTC), plus on push to `main`
- **Build**: Run `npm run build` (Next.js static export)
- **Deploy**: Publish the output to Cloudflare Pages, serving at http://marketplace.centy.io/

## Implementation Notes

### Workflow file
Create `.github/workflows/deploy.yml` with:
- `schedule` trigger: `cron: '0 6,18 * * *'`
- `push` trigger on `main` branch
- Steps:
  1. Checkout repo
  2. Setup Node.js
  3. Install dependencies (`npm ci`)
  4. Build project (`npm run build`)
  5. Deploy to Cloudflare Pages using [`cloudflare/pages-action`](https://github.com/cloudflare/pages-action)

### Required GitHub Secrets
The following secrets must be added to the repository settings:
- `CLOUDFLARE_API_TOKEN` — Cloudflare API token with Pages edit permissions
- `CLOUDFLARE_ACCOUNT_ID` — Cloudflare account ID

### Cloudflare Pages config
- **Project name**: match existing CF Pages project name
- **Build output directory**: `out` (Next.js static export)
- **Custom domain**: `marketplace.centy.io`

### next.config.ts
Ensure `output: 'export'` is set for static site generation.

## Acceptance Criteria

- [x] Workflow runs on schedule (twice daily) and on push to `main`
- [x] Build succeeds
- [x] Deployment reaches Cloudflare Pages
- [x] Site is accessible at http://marketplace.centy.io/
- [x] Required secrets are documented
